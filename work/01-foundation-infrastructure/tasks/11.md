## Description

Acceptance testing for the 01-foundation-infrastructure feature. All implementation tasks (1–7) and audit tasks (8–10) are complete. The QA agent runs a full acceptance check: executes the Gradle build with all three integration tests, starts the backend, and verifies every acceptance criterion from user-spec and tech-spec using bash, curl, and docker commands. No deployment required — all checks run locally.

The QA covers: build correctness, security config behavior (no 401 on /health), folder structure, .env.example contents, docker-compose ports, .gitignore completeness, and scripts/install-hooks.sh executability. Frontend verification (Nuxt page at :3000) is delegated to the user.

## What to do

Execute all 8 verification steps from the Agent Verification Plan:

1. Run `cd backend && ./gradlew build` — confirm output is `BUILD SUCCESSFUL` and all three integration tests pass: `HealthEndpointTest`, `HealthSecurityTest`, `SecurityBlockTest`.

2. Start backend in background: run `./gradlew bootRun` from the `backend/` directory, wait until Netty startup log appears (look for "Netty started on port 8080").

3. Run `curl -s http://localhost:8080/health` — confirm response body is `{"status":"ok"}` and HTTP status is 200.

4. Run `curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/health` without any auth header — confirm result is `200`. This verifies SecurityConfig correctly permits `/health` without authentication.

5. Run `ls backend/ frontend/ infra/ docs/` — confirm all four directories exist in the repo root.

6. Run `grep -E "MONGODB_URI|REDIS_URL" .env.example` — confirm both variables are present.

7. Run `docker compose -f infra/docker-compose.yml up -d` — confirm both MongoDB and Redis services start without errors.

8. Run `nc -z localhost 27017 && nc -z localhost 6379` — confirm both ports are accessible.

9. Run `curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/api/nonexistent` — confirm result is `401`. This verifies `anyExchange().authenticated()` blocks undefined paths at runtime (not just in code inspection).

After all steps pass, write the QA report to `work/01-foundation-infrastructure/logs/working/task-11/qa-report.md`.

- Named volumes `mongodb_data` and `redis_data` must be defined
