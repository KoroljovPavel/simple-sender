## Details

**Files:**
- `docs/local-setup.md` (new) ‚Äî the complete onboarding guide

**Required sections and content:**

1. **Prerequisites**
   - Java 21 JDK (not JRE) ‚Äî link to adoptium.net or similar; note: Gradle toolchain can auto-provision via Foojay if JDK is not installed, but JDK is still required for IntelliJ
   - Node 24 LTS ‚Äî reference `.nvmrc`; recommend installing via nvm (`nvm install` in the repo root reads `.nvmrc` automatically)
   - Docker Desktop (macOS/Windows) or Docker Engine (Linux)
   - pnpm ‚Äî install via `npm install -g pnpm` or `corepack enable && corepack prepare pnpm@latest --activate`
   - gitleaks ‚Äî `brew install gitleaks` (macOS); verify with `gitleaks version`; note it is optional but strongly recommended (hook warns but does not block if missing)

2. **Step-by-step setup** (must match this exact order and commands from tech-spec):
   - Step 1: `bash scripts/install-hooks.sh` ‚Äî copies pre-commit hook to `.git/hooks/pre-commit`
   - Step 2: `cp .env.example .env` ‚Äî defaults work for local dev without changes
   - Step 3: `docker compose -f infra/docker-compose.yml up -d` ‚Äî starts MongoDB on :27017 and Redis on :6379
   - Step 4: `cd backend && ./gradlew bootRun` ‚Äî Netty starts on :8080
   - Step 5: `GET http://localhost:8080/health` ‚Äî verifies backend is up (`{"status":"ok"}`)
   - Step 6: `cd frontend && pnpm install && pnpm dev` ‚Äî Nuxt dev server starts on :3000

3. **Verification checklist:**
   - `curl http://localhost:8080/health` ‚Üí `{"status":"ok"}` with HTTP 200
   - Open `http://localhost:3000` in browser ‚Äî Hello World page loads without console errors

4. **Port conflict resolution** (from tech-spec Risks):
   - MongoDB :27017 ‚Üí change host port in `infra/docker-compose.yml` (the `ports:` mapping, left side)
   - Redis :6379 ‚Üí same, change host port in `infra/docker-compose.yml`
   - Backend :8080 ‚Üí change `server.port` in `backend/src/main/resources/application.properties`
   - Frontend :3000 ‚Üí change `devServer.port` in `frontend/nuxt.config.ts`

5. **Windows/WSL2 warning:**
   - Bash shell scripts (`scripts/install-hooks.sh`) require a Unix-compatible shell
   - Docker Compose and shell commands may behave differently on native Windows
   - Recommendation: use WSL2 on Windows; all commands in this guide assume Linux/macOS or WSL2

**Dependencies:** Tasks 1‚Äì6 must be complete before writing this doc, so commands and variable names can be verified against the actual files.

**Edge cases:**
- `.env.example` may contain inline comments ‚Äî preserve them in any `cp` example in the doc
- `./gradlew bootRun` on first run downloads Gradle and all dependencies ‚Äî this can take several minutes; mention it
- If gitleaks is not installed, the hook exits 0 with a warning ‚Äî document this behavior so developers understand why the hook does not block commits
