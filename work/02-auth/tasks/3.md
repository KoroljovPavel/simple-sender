# Task 3: Email service + templates

## Required Skills

Before starting, load:
- `/skill:code-writing` — [SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Create `EmailService` wrapping `JavaMailSender` with a fire-and-forget reactive pattern so email failures never propagate to the caller. Create three plain-HTML email templates in Ukrainian as classpath resources. HTML-escape user-supplied data to prevent XSS in emails.

This task is in Wave 2 alongside Task 4. Both Task 5 (registration) and Task 6 (password reset) depend on `EmailService` being available. Task 3 has no TDD complexity — unit tests mock `JavaMailSender` and verify the correct call was made.

## What to do

1. **Create `EmailService`** at `backend/src/main/java/com/botfunnel/email/EmailService.java`:
   - Constructor-inject `JavaMailSender`, `@Value("${app.mail.from}") String from`, `@Value("${app.url}") String appUrl`
   - Three public methods: `sendVerificationEmail(String to, String name, String token)`, `sendPasswordResetEmail(String to, String name, String token)`, `sendAccountBlockedEmail(String to, String name, String supportEmail)`
   - Each method: create `MimeMessage` via `javaMailSender.createMimeMessage()`, use `MimeMessageHelper(msg, true, "UTF-8")` for HTML
   - Load template from classpath (`getClass().getResourceAsStream(...)`), replace placeholders via `String.replace()`
   - HTML-escape `name` before embedding: replace `&` → `&amp;`, `<` → `&lt;`, `>` → `&gt;`, `"` → `&quot;`
   - Fire-and-forget pattern: `Mono.fromCallable(() -> { sendMsg(); return null; }).subscribeOn(Schedulers.boundedElastic()).subscribe(null, err -> log.error("Email send failed to {}: {}", to, err.getMessage()))`

2. **Create email templates** at `backend/src/main/resources/templates/email/`:
   - `verify-email.html` — subject "Підтвердіть email", body with personalized greeting, verification link `{APP_URL}/auth/verify-email?token={TOKEN}`, expiry note (24 годин)
   - `reset-password.html` — subject "Скидання пароля", body with link `{APP_URL}/auth/reset-password?token={TOKEN}`, expiry note (1 година), note about ignoring if not requested
   - `account-blocked.html` — subject "Ваш акаунт заблоковано", body with reason placeholder and support contact `{SUPPORT_EMAIL}`
   - All three templates: valid HTML, inline CSS only, Ukrainian text

**Dependencies:**
Task 1 (Gradle deps, Mailpit docker-compose), Task 2 (UserRepository for EmailService wiring if needed).

**Decision 10 (plain HTML, no Thymeleaf):** Do NOT add `spring-boot-starter-thymeleaf` to `build.gradle`. Use `String.replace()` with named placeholders like `{TOKEN}`, `{NAME}`, `{APP_URL}`, `{SUPPORT_EMAIL}`.

**Edge cases:**
- Template file not found: `getClass().getResourceAsStream()` returns null if path wrong. Log a clear error and return early.
- `MimeMessageHelper` charset: use `UTF-8` explicitly to support Ukrainian characters
- `Schedulers.boundedElastic()` is correct for blocking I/O like SMTP; do NOT use `Schedulers.parallel()`

**Implementation hints:**
- HTML-escape function: simple `String.replace()` chain for 4 chars is sufficient; no external library needed
- Template path in classpath: `"/templates/email/verify-email.html"` (leading slash, classpath root)
- In unit tests: use `@MockitoBean` (`org.springframework.test.context.bean.override.mockito.MockitoBean`) for `JavaMailSender`
- For fire-and-forget timing: the `.subscribe()` call returns immediately; the actual send happens asynchronously; use `Awaitility.await().atMost(2, SECONDS).until(...)` to verify async email was sent — `Thread.sleep()` is fragile in test contexts. Awaitility is available via `spring-boot-starter-test` transitive dependency.
