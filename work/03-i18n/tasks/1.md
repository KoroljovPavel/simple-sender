# Task 1: Playwright bootstrap

## Required Skills

Перед виконанням задачі завантаж:

## Description

Бутстрапимо Playwright як E2E test runner у `frontend/`. Зараз `@playwright/test` у проекті відсутній — потрібно встановити пакет як devDependency, створити `playwright.config.ts` з webServer-конфігом (Nuxt dev на порту 3000), додати npm-script `test:e2e`, створити порожню директорію `frontend/e2e/` (через `.gitkeep`), створити/оновити `frontend/.gitignore` так, щоб артефакти Playwright (`playwright-report/`, `test-results/`) не комітились.

Задача виконується solo у Wave 1 (per Decision 17), бо Task 2 (i18n module bootstrap) теж модифікує `frontend/package.json` — паралельне виконання дало б merge-конфлікт у dependencies/scripts. Серіалізація через окремі Waves — найпростіша мітигація. На цьому етапі E2E-сценарії НЕ пишемо: реальна i18n-спека (`e2e/i18n.spec.ts`) додається у Wave 5 (Task 11), коли pages вже локалізовані. Завдання Wave 1 — лише інфраструктура: `pnpm test:e2e --list` має успішно стартувати і повернути "0 tests found" або еквівалент (config валідний, директорія існує).

## What to do

1. Встановити `@playwright/test@^1.49` як devDependency у `frontend/package.json` (через `pnpm add -D @playwright/test@^1.49` запущений з `frontend/`). Lockfile (`frontend/pnpm-lock.yaml`) оновлюється автоматично.
2. Створити `frontend/playwright.config.ts` з мінімально достатньою конфігурацією:
   - `testDir: './e2e'`
   - `use.baseURL: 'http://localhost:3000'`
   - `webServer` блок: `command: 'pnpm dev'`, `url: 'http://localhost:3000'`, `reuseExistingServer: !process.env.CI`, розумний `timeout` (наприклад 120_000) щоб Nuxt dev встиг піднятись
   - `reporter: 'list'` (мінімалістично; можна `'html'` за бажанням, але артефакти у `playwright-report/` мають ігноруватись)
   - один `projects` запис (наприклад chromium з `devices['Desktop Chrome']`) — достатньо для подальшого i18n.spec.ts
3. Додати npm-script `"test:e2e": "playwright test"` у `frontend/package.json:scripts`. Існуючі scripts (`dev`, `build`, `generate`, `preview`, `test`) не чіпаємо.
4. Створити директорію `frontend/e2e/` з placeholder-файлом `frontend/e2e/.gitkeep` (порожній), щоб git зафіксував директорію.
5. Створити `frontend/.gitignore` (зараз файлу немає — ігнори frontend живуть у root `.gitignore`). Додати у новий `frontend/.gitignore`:
   ```
   playwright-report/
   test-results/
   ```
   Файл — інкрементально-додатковий до root `.gitignore`, не дублює і не переписує його правила.
6. НЕ запускати `pnpm exec playwright install` (бінарі браузерів) у рамках цієї задачі — вони підтягуються при першому `pnpm test:e2e` локально. Verify-smoke `--list` не вимагає браузерів.

## Verification Steps

### Smoke

Виконуються агентом одразу після implementation, до review:

1. `cd frontend && pnpm exec playwright --version` → друкує версію Playwright (наприклад `Version 1.49.x`), exit 0.
2. `cd frontend && pnpm test:e2e --list` → output на кшталт "Total: 0 tests in 0 files" або еквівалент (config валідний, директорія `e2e/` існує). Не повинно бути помилок про missing config / missing testDir.
3. `cd frontend && cat package.json | grep -A1 '"scripts"'` → містить `"test:e2e"`.
4. `cd frontend && cat .gitignore` → містить `playwright-report/` і `test-results/`.
5. `git status` → нові файли (`frontend/playwright.config.ts`, `frontend/e2e/.gitkeep`, `frontend/.gitignore`) у "untracked" або "to be committed"; артефакти Playwright (якщо випадково створились локально) у `git status` НЕ з'являються.

## Details

**Files:**
- `frontend/package.json` — додати `@playwright/test@^1.49` у `devDependencies`; додати `"test:e2e": "playwright test"` у `scripts`. Зараз файл містить scripts {dev, build, generate, preview, test} і devDependencies {@nuxt/test-utils, @nuxtjs/tailwindcss, @vue/test-utils, happy-dom, shadcn-vue, vitest}. `packageManager: pnpm@10.33.2` зафіксований — використовуй той же pnpm.
- `frontend/pnpm-lock.yaml` — оновлюється автоматично командою `pnpm add -D`. Не редагувати вручну.
- `frontend/playwright.config.ts` (new) — `defineConfig` з `testDir`, `use.baseURL`, `webServer`, `projects`, `reporter`. TypeScript file (Nuxt уже у ts-режимі).
- `frontend/e2e/.gitkeep` (new) — порожній файл, маркер директорії для git.
- `frontend/.gitignore` (new) — два рядки: `playwright-report/`, `test-results/`. Root `.gitignore` уже ігнорує `frontend/node_modules/`, `frontend/.nuxt/`, `frontend/.output/`, `frontend/dist/` — не дублюй.

**Dependencies:** жодних задач-залежностей. Solo Wave 1.

**Edge cases:**
- `pnpm add -D @playwright/test@^1.49` може встановити мінорний/патч-апгрейд. Це OK, поки major лишається 1.x.
- `webServer` у Playwright config: якщо `reuseExistingServer: false` у CI, Playwright форсить старт нового dev-сервера. Прийнятно.
- `--list` без браузерних бінарей: Playwright `--list` НЕ запускає браузер, лише парсить config і збирає тести. Має працювати без `playwright install`.
- `frontend/.gitignore` як новий файл: НЕ дублюй правила з root `.gitignore` (`node_modules/`, `.nuxt/`, etc.) — тримай мінімально, тільки Playwright-артефакти. Це уникає divergence між двома gitignore при майбутніх змінах.

**Implementation hints:**
- Перед `pnpm add` перевір `which pnpm` і версію (має бути ~10.33.2 per `packageManager` field).
- Запускай команди у `frontend/` через `cd frontend && ...` — pnpm читає `package.json` з cwd.
- При ручному редагуванні `package.json` зберігай порядок ключів (скрипти top-down: dev, build, generate, preview, test, test:e2e). JSON-форматування — 2 spaces (як у існуючому файлі).
- `playwright.config.ts` — імпорт `import { defineConfig, devices } from '@playwright/test';`. Не використовуй `require`.
- Якщо `pnpm test:e2e --list` падає на missing webServer port — це OK у тестовому контексті: `--list` не стартує webServer. Перевіряй що config парситься без exception.
