# Task 3: Server middleware — strip invalid/default-locale prefix

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](../../../.claude/skills/code-writing/SKILL.md)

## Description

Створюємо Nitro global middleware `frontend/server/middleware/strip-invalid-locale.ts`, який ловить будь-який невалідний 2-літерний locale-префікс у URL (наприклад `/fr/...`) **і** заборонений під strategy `prefix_except_default` префікс default-локалі (`/uk/...`) → виставляє 302 на path без префікса. Виконується до Vue Router і до auth-middleware, забезпечуючи стабільний редирект незалежно від auth-стану.

> **Wave deviation note:** tech-spec (lines 285, 295) групує цю таску у Wave 2 паралельно з Task 2. Ми переносимо її у Wave 3 з `depends_on: [2]`, бо middleware імпортує константу `NON_DEFAULT_LOCALES`, яку створює Task 2 — реальна import-залежність робить справжню паралельність неможливою. Це навмисне відхилення від wave-grouping тех-спека для execution safety (запобігаємо merge-конфлікту і race на створенні `frontend/shared/i18n-locales.ts`).

Закриває AC7 з user-spec/tech-spec, спирається на Decision 5 (нормалізація path для захисту від open-redirect, headers `X-Robots-Tag`/`Referrer-Policy` на 302-response для auth-zone щоб не лікав `?token=` через Referer). Allow-list `['en']` НЕ хардкодиться у middleware-файлі — імпортується з `NON_DEFAULT_LOCALES` константи (визначеної у Task 2 — спільне джерело правди для middleware і `routeRules`, щоб додавання 3-ї локалі не вимагало синхронізації у двох місцях).

Decision-логіку (parse → match → normalize → decide redirect) декомпозуємо у окрему чисту функцію без залежності від `H3Event`, щоб вона тестувалась unit-тестами без mocking Nitro internals. Сам middleware-handler — тонкий wrapper, який викликає чисту функцію і робить `sendRedirect` / `setResponseStatus` / `setHeaders` через `h3` API.

## What to do

1. Створити чисту exportable функцію (наприклад `decideLocaleStrip(path: string, nonDefaultLocales: readonly string[])`) у тому самому модулі (або окремому helper-модулі — на розсуд агента, головне щоб тестувалась без `H3Event`). Функція повертає одне з трьох рішень:
   - **passthrough** — path не починається з 2-літерного префікса, або префікс — валідна не-default локаль (`en`).
   - **reject** (HTTP 400, no redirect) — path містить backslash (`\`).
   - **redirect** (HTTP 302) — префікс невалідний (`fr`) або default-locale (`uk`); функція повертає нормалізований target path (з query string preserved), і прапор `isAuthZone: boolean` для додавання headers.
2. Регекс `^/([a-z]{2})(?=/|$)` витягує potential locale-префікс. Lookahead `(?=/|$)` важливий — щоб не зматчити `/enabled` як `en` + `abled`.
3. Path normalization order (per Decision 5):
   - **(a) reject backslash** перш за все — не нормалізуй, не декодуй, поверни 400.
   - **(b) strip prefix** (видалити `/{xx}` з початку, лишити `/...` або `/`).
   - **(c) collapse leading slashes** — `^/+` → `/` (щоб `/fr//evil.com` після strip → `//evil.com` не став protocol-relative URL → нормалізується на `/evil.com`).
   - **(d) decode percent-encoding** — **після** normalization, щоб double-encoded payload `%2F%2Fevil.com` не обходив collapse-step. Якщо decoding кидає (`URIError` на malformed sequences) — поверни reject 400.
4. Auth-zone match — `normalizedPath.startsWith('/auth')` БЕЗ trailing slash (ловить `/auth?token=x` і `/auth/verify-email`, але не `/authentic` — тут /auth завжди префікс шляху, інших route-ів з таким початком немає; якщо хочеш бути ще консервативнішим — `=== '/auth' || startsWith('/auth/') || startsWith('/auth?')`).
5. Middleware-handler:
   - Викликає decision-function з `event.path` і імпортованим `NON_DEFAULT_LOCALES`.
   - На `passthrough` — нічого не робить (return).
   - На `reject` — `setResponseStatus(event, 400)`, `return 'Bad Request'` (or empty body — без redirect).
   - На `redirect` — якщо `isAuthZone`, спочатку `setResponseHeaders(event, { 'X-Robots-Tag': 'noindex, nofollow', 'Referrer-Policy': 'no-referrer' })`, потім `sendRedirect(event, targetPath, 302)`. Headers ставляться на 302-response, не на target — це і є мітигація проти Referer leak (`?token=` не попадає у Referer browser-side).
6. Зберегти query string і fragment у redirect target. `event.path` у Nitro містить query — якщо ти парсиш його як URL, не загуби `?...`. Найпростіше — окремо обробляти `pathname` (для regex-матчу і normalization) і додавати `query` назад у `targetPath` перед redirect.
7. Створити `frontend/tests/server/strip-invalid-locale.spec.ts` (Vitest) **до** імплементації middleware (TDD). Тестуємо саме чисту decision-function, не H3 handler — fixture cases per Testing Strategy:
   - `/fr/dashboard` → redirect `/dashboard`, isAuthZone=false.
   - `/uk/dashboard` → redirect `/dashboard`, isAuthZone=false (default-locale prefix теж стрипається).
   - `/en/dashboard` → passthrough.
   - `/dashboard` → passthrough.
   - `/fr/auth/verify-email?token=abc` → redirect `/auth/verify-email?token=abc`, isAuthZone=true.
   - `/fr//evil.com` → redirect `/evil.com` (collapse leading slashes після strip).
   - `/fr/\\evil.com` (path містить backslash) → reject (400).
   - Запустити тести → переконатись що падають. Реалізувати функцію → тести проходять.
8. Запустити smoke-checks (див. Verification Steps → Smoke) на live `pnpm dev`.

- `tests/server/strip-invalid-locale.spec.ts::default-locale prefix /uk/dashboard redirects to /dashboard` — `/uk/dashboard` → redirect `/dashboard` (бо `prefix_except_default` забороняє `uk`-префікс; allow-list = `['en']`).
- `tests/server/strip-invalid-locale.spec.ts::valid en prefix passthrough` — `/en/dashboard` → `{ kind: 'passthrough' }`.
- `tests/server/strip-invalid-locale.spec.ts::no prefix passthrough` — `/dashboard` → passthrough.
- `tests/server/strip-invalid-locale.spec.ts::auth-zone redirect preserves query and flags isAuthZone` — `/fr/auth/verify-email?token=abc` → `{ kind: 'redirect', target: '/auth/verify-email?token=abc', isAuthZone: true }`.
- `tests/server/strip-invalid-locale.spec.ts::collapse leading double slashes after strip` — `/fr//evil.com` → redirect `/evil.com` (захист від protocol-relative redirect).
- `tests/server/strip-invalid-locale.spec.ts::backslash in path is rejected` — `/fr/\\evil.com` → `{ kind: 'reject', status: 400 }`.
- `tests/server/strip-invalid-locale.spec.ts::regex lookahead does not match longer words` — `/enabled` → passthrough (НЕ зматчити як `/en` + `/abled`).
- `tests/server/strip-invalid-locale.spec.ts::malformed percent-encoding rejected` — `/fr/%E0%A4%A` (incomplete sequence) → reject (decodeURI throws → 400).
- `tests/server/strip-invalid-locale.spec.ts::test_percent_encoded_double_slash` — `/fr/%2Fevil.com` після strip+normalize → редирект НЕ продукує `//evil.com` (path normalization happens BEFORE percent decode per Decision 5; після strip path = `/%2Fevil.com`, collapse `^/+` не зачіпає `%2F`, decode → `//evil.com`, тоді safety-net collapse → `/evil.com`). Security-critical: захист від open-redirect через encoded slash.
- `tests/server/strip-invalid-locale.spec.ts::test_percent_encoded_backslash` — `/fr/%5Cevil.com` (backslash percent-encoded) → handled safely: після decode path містить `\` → reject (400). Перевіряємо що encoding не обходить backslash-reject.
- `tests/server/strip-invalid-locale.spec.ts::test_double_encoded_payload` — `/fr/%252F%252Fevil.com` (double-encoded `%2F%2F`) → не bypass: per Decision 5 decode виконується один раз ПІСЛЯ collapse, тому перший decode дає `%2F%2Fevil.com`, safety-net collapse після decode не зачіпає `%2F`-літерали — фінальний target НЕ містить literal `//`. Перевіряємо що double-encoded payload не дає protocol-relative URL.

**Dependencies:**

- Vitest вже сконфігурований у проекті (per code-research §11) — `@nuxt/test-utils/config`, `defineVitestConfig`. Spec-файл просто розміщується у `frontend/tests/server/` (нова піддиректорія), Vitest автоматично її підхоплює.

**Edge cases:**
- `/fr` (рівно 2 літери, без trailing slash і без шляху далі) — regex `^/([a-z]{2})(?=/|$)` матчить через `(?=$)`. Після strip path → `/` (root). Redirect на `/`.
- `/fR/dashboard` (uppercase) — regex `[a-z]{2}` НЕ матчить (case-sensitive). Passthrough. Це ок, бо `@nuxtjs/i18n` теж не зматчить uppercase locale.
- `/enabled` — `/en` могло б зматчити, але lookahead `(?=/|$)` блокує (наступний char — `a`, не `/` і не end-of-string). Passthrough.
- Query-only request `/?foo=bar` — regex не матчить (немає 2-літерного префікса після `/`). Passthrough.
- `/fr/auth` (auth без trailing slash, без query) — startsWith('/auth') true, isAuthZone=true, headers ставляться. Це бажана поведінка.
- Path з encoded forward-slash `/fr/%2Fevil.com` — після strip path = `/%2Fevil.com`, потім collapse `^/+` → `/` (один слеш, бо `%2F` ще не decoded), потім decode → `//evil.com`. **РИЗИК open-redirect**, бо decode після collapse. Mitigation: після decode перевірити ще раз чи path починається з `//` і повторити collapse, АБО reject decoded path якщо містить leading `//`. Декомпозуй це у функцію — не пропусти кейс у тестах. Альтернативний підхід: decode ПЕРЕД collapse, тоді кейс `%2F%2Fevil.com` зловиться. Але тоді кейс double-encoded `%252F%252Fevil.com` → перший decode дає `%2F%2F...`, ще один проход — `//...`. Decision 5 каже "decode after normalization to avoid decode-then-match bypass" — отже, decode один раз, потім ще раз collapse як safety net. Покриваємо тестами обидва кейси (`%2F%2F` і `%252F%252F`).
- Multiple consecutive backslashes `\\\\evil.com` після `/fr/` — backslash reject спрацює на першому символі (рекомендовано перевіряти naked path до будь-яких трансформацій).
- **Nitro `~/` alias resolution** — alias `~/` може бути не сконфігурований за замовчуванням у server middleware context Nuxt 4. Verify import resolution під час `pnpm dev` smoke: якщо `import ... from '~/shared/...'` кидає module-not-found у Nitro logs — переходь на relative path (`../../shared/i18n-locales`) як fallback. Не блокує реалізацію, але важливо валідувати у smoke-checks.

**Implementation hints:**
- Nitro Nitro global middleware у Nuxt лежить у `frontend/server/middleware/<name>.ts` і автоматично підвантажується. Експортуй `export default defineEventHandler(async (event) => { ... })`.
- API: `getRequestURL(event)` повертає повний URL з query, але простіше парсити з `event.path` (містить pathname + search).
- `import { sendRedirect, setResponseStatus, setResponseHeaders } from 'h3'` — ці функції доступні в Nitro out of the box.
- Імпорт `NON_DEFAULT_LOCALES` — Nitro alias `~/` може бути не сконфігурований за замовчуванням для server middleware у Nuxt 4 (auto-imports працюють для `composables/`, `utils/`, але резолюшен `~/shared/...` під Nitro context треба перевірити у dev-сервері). Стратегія: спочатку спробуй `import { NON_DEFAULT_LOCALES } from '~/shared/i18n-locales'`; якщо `pnpm dev` падає на module-not-found — перейди на relative path: `import { NON_DEFAULT_LOCALES } from '../../shared/i18n-locales'` (з `frontend/server/middleware/strip-invalid-locale.ts` до `frontend/shared/i18n-locales.ts`). Перевір актуальний шлях у Task 2 — чи константа лежить у `frontend/shared/i18n-locales.ts`, чи re-export з `nuxt.config.ts`.
- Для unit-тестів decision-function — звичайний `it('...', () => { expect(decideLocaleStrip('/fr/dashboard', ['en'])).toEqual({ kind: 'redirect', target: '/dashboard', isAuthZone: false }) })`. Без `mountSuspended`, без `mockNuxtImport` — це чиста функція.
- `decodeURIComponent` кидає `URIError` на malformed sequences — обгорни `try/catch` і поверни reject.
- Не міксуй `decodeURI` (зберігає reserved chars типу `/`) і `decodeURIComponent` (декодує все). Тут потрібен `decodeURIComponent` бо ми хочемо побачити `%2F` як `/`.
