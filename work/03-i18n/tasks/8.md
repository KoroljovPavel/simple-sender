## Description

Wire up i18n at the application shell layer: `frontend/app.vue`, `frontend/layouts/default.vue`, and `frontend/layouts/auth.vue`.

`app.vue` becomes the single place that drives `<html lang>` and `hreflang` alternates via `useLocaleHead({ seo: true })` + `useHead` injection â€” required so SSR emits the right `lang` attribute on first paint and search engines see the proper alternates for both UA and EN URLs.

`layouts/default.vue` (the authenticated shell) currently hardcodes `Bot Funnel` (brand), `Logout`, `Dashboard`, `Profile` and uses raw `<NuxtLink>` + `navigateTo('/auth/login')`. All visible strings are routed through `t()` â€” brand via `t('brand.name')` per Decision 13 (same value `"Bot Funnel"` in both locales but kept under a key so AC2 holds). `<NuxtLink>` is replaced with `<NuxtLinkLocale>`, the logout `navigateTo` is wrapped in `localePath()`, and `<LangSwitcher />` is added to the topbar to the right of the Logout button in the same flex-row (per Decision 15).

`layouts/auth.vue` (the centered-card shell used by all auth pages) currently has zero visible text but no LangSwitcher either. Per Decision 15, place `<LangSwitcher />` as `position: absolute; top: 1rem; right: 1rem; z-index: 10` over the centered card so unauthenticated visitors on `/auth/login`, `/auth/register`, etc. can switch language without scrolling or hunting.

This task does **not** translate the auth pages or app pages themselves â€” those land in Tasks 9 and 10. Scope here is strictly the shell. After this task, opening `/dashboard` shows a localized topbar + LangSwitcher; opening `/auth/login` shows a LangSwitcher in the top-right corner over the card.

**Edge cases:**

- **`useLocaleHead` SSR vs hydration.** v9 returns reactive refs; `useHead` should consume them directly. Do not wrap in `computed(() => head.value)` â€” it breaks reactivity. Confirm by SSR-rendering both `/dashboard` and `/en/dashboard` and inspecting `<html lang>` in raw HTML (curl, not DevTools â€” DevTools shows post-hydration state).
- **`<LangSwitcher />` in `auth.vue` over small viewports.** At â‰¤ 360px the centered card may overflow horizontally; the absolute switcher should not collide. Tailwind `top-4 right-4` (1rem each) keeps it clear of card padding.
- **Brand wordmark.** Per Decision 13, `brand.name` returns `"Bot Funnel"` in both locales â€” never localized. Do not introduce a translated variant; if a future requirement needs one, it lands as a separate decision update.
- **Logout redirect locale persistence.** `localePath('/auth/login')` on `uk` returns `/auth/login`; on `en` returns `/en/auth/login`. The user stays in the same locale after logout â€” desired UX.
- **`<NuxtLinkLocale>` semantics.** Pass the `to` attribute as the **unprefixed** path (e.g., `to="/dashboard"`); the module injects the locale prefix. Do not write `to="/en/dashboard"` manually â€” it would double-prefix on `en`.
- **No layout for unauthenticated users hitting `/auth/login`.** `auth.vue` is the layout for `/auth/*` pages (set via `definePageMeta({ layout: 'auth' })` on each page) â€” the LangSwitcher appearing here means it works pre-login, which is required by user-spec (LangSwitcher reachable from any page).

**Implementation hints:**

- Follow the v9 docs pattern for SEO head:
  ```ts
  const head = useLocaleHead({ seo: true })
  useHead(head)
  ```
  Both come from `@nuxtjs/i18n` auto-imports; no manual import needed.
- For `default.vue` topbar flex-row, the existing class `flex items-center gap-3` already handles spacing â€” `<LangSwitcher />` slots in as the last child without extra wrapping.
- For `auth.vue`, the existing `min-h-screen flex items-center justify-center` already implies block-level positioning; add `relative` to the root so absolute children anchor there. If `<LangSwitcher />` does not pass through the `class` attribute, wrap it in a `<div class="absolute top-4 right-4 z-10">` instead.
- Do not introduce extra wrapper `<div>`s in `default.vue` topbar â€” the current `flex` row holds three direct children today (user info span, logout button, and after this change LangSwitcher) and that is fine.
- Verify `useLocaleHead` output by curling raw SSR HTML: `curl -s http://localhost:3000/dashboard | grep -E '<html|hreflang'` and `curl -s http://localhost:3000/en/dashboard | grep -E '<html|hreflang'`.
