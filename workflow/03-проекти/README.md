# 03 — Проекти (Workspaces)

## Мета
Дати кожному користувачу можливість створити ізольований простір (проект), у якому будуть свої боти, підписники, воронки, розсилки, налаштування.

## Користувацька цінність
Користувач може вести декілька різних бізнесів / брендів окремо в одному акаунті, не змішуючи аудиторію та сценарії.

---

## Концепція
Проект — це **isolation boundary**. Усе, що відбувається всередині (підписники, воронки, ключі), належить тільки одному проекту і не змішується з іншими. Жоден API-запит не обходиться без `project_id` у контексті.

---

## MUST HAVE

### CRUD проектів
- **Створити проект:**
  - Назва (3-50 символів, обов'язково)
  - Короткий опис (опц., до 200 символів)
  - Часовий пояс (для розкладу розсилок) — за замовчуванням з браузера
- **Перейменувати проект** (назва, опис, timezone)
- **Видалити проект:**
  - Підтвердження з вводом назви проекту (anti-mistake захист)
  - Soft-delete на 7 днів (можна відновити)
  - Через 7 днів — hard-delete каскадно: бот, підписники, воронки, розсилки, ключі, події
- **Список своїх проектів** на головній після логіну

### Доступ і authorization
- Owner = creator. Усі права у власника.
- Для MVP — без додаткових учасників (співавтори в `12-nice-to-have`)
- **Перевірка доступу на КОЖНОМУ запиті:** middleware/guard перевіряє, що поточний User є Owner запитуваного Project
- Усі URL мають `project_id`: `/api/v1/projects/{projectId}/subscribers`
- Без project_id у URL — 400 Bad Request, ніколи не "вгадуємо" поточний проект з сесії на бекенді

### Перемикання контексту в UI
- Селектор поточного проекту вгорі сторінки (dropdown)
- Усі сторінки (підписники, воронки, розсилки) фільтровані по поточному проекту
- При відсутності проектів — onboarding-сторінка "Create your first project"
- Останній обраний проект пам'ятається в localStorage

### Сторінка налаштувань проекту
- Загальні: назва, опис, timezone
- Підключення бота → детально в `04-телеграм-боти`
- Custom fields → детально в `05-підписники`
- API key → детально в `08-публічне-api`
- Webhook URL → детально в `08-публічне-api`
- Зона небезпеки: видалення проекту

### Ліміти (anti-abuse)
- До 5 проектів на користувача (м'який ліміт, константа в конфізі)
- При досягненні ліміту — UI блокує кнопку "Create" з поясненням

---

## Що НЕ входить (свідомо)
- Запрошення співавторів — `12-nice-to-have`
- Ролі всередині проекту (admin/editor/viewer) — `12-nice-to-have`
- Тарифні плани з різними лімітами — `12-nice-to-have`
- Перенос проекту іншому власнику — `12-nice-to-have`
- Архівування проектів (без видалення) — `12-nice-to-have`
- Дублювання проекту — `12-nice-to-have`

---

## Acceptance criteria
1. Користувач створює проект і одразу бачить його у списку
2. Користувач не бачить чужих проектів навіть якщо вгадає URL з ID (повертається 403)
3. Видалення проекту прибирає весь зв'язаний контент через 7 днів
4. Не можна створити більше 5 проектів — UI блокує, бекенд теж
5. Перемикач проектів працює і запам'ятовує останній обраний

---

## Залежності
- `02-автентифікація` (тільки залогінений user може створити проект)

---

## Технічні нотатки
- На таблиці `projects` додати колонку `deleted_at` для soft-delete
- Cron-job раз на добу: видаляти hard все з `deleted_at < now() - 7 days`
- Усі дочірні таблиці (subscribers, funnels, etc.) мають `project_id` з FK і CASCADE
- Індекси на `project_id` в кожній дочірній таблиці — обов'язково (швидкі фільтрації)
